# AGENTS.md

## Контекст
- Название проекта: `Календарь звонков`.
- Общение с пользователем и содержимое `AGENTS.md` вести на русском языке.
- Проект учебный: по возможности работу выполнять через агентов, а не вручную.

## Текущее состояние репозитория
- Репозиторий пока минимальный: в корне есть только `README.md` и `.github/workflows/`.
- `README.md` сейчас содержит только badge и не описывает запуск или архитектуру.
- В репозитории пока нет `package.json`, lockfile, workspace-конфига и локальных команд сборки/тестов. Не придумывать команды до появления реальных конфигов.

## Проверенные источники истины
- Основной проверенный CI-факт: `.github/workflows/hexlet-check.yml` запускается на каждый `push`.
- `.github/workflows/README.md` прямо говорит: `hexlet-check.yml` нельзя удалять или редактировать, и репозиторий нельзя переименовывать.

## Практические правила для агентов
- Перед любыми выводами о стеке сначала перечитывать текущие конфиги и manifests: на этом этапе структура проекта еще не сформирована.
- Если в репозитории по-прежнему нет исполняемых конфигов, опираться на актуальную задачу пользователя и фиксировать решения в артефактах проекта, не выдавая предположения за существующую инфраструктуру.
- Так как репозиторий подключен к GitHub, коммиты допустимы только по явной просьбе пользователя.
- Если обычный `git push origin <branch>` по HTTPS не проходит из-за отсутствия интерактивной авторизации, но `gh auth status` показывает активную сессию, допустимо использовать одноразовый push через временный URL вида `https://x-access-token:$(gh auth token)@github.com/<owner>/<repo>.git` без изменения `git remote` и `git config`.
